    \item (Step 5) Draw the complete block diagram of your first version implementation according to the Farrow structure. (5\%) Show your hardware sharing version (the precision of the datapath must be BF16 and cannot be changed). (10\%) Draw the block diagram after your improvement. (5\%) Indicate the critical path. (5\%)

    Figure~\ref{fig:first_ver} shows the first version block diagram following the Farrow structure. The three input samples $x(m)$, $x(m+1)$, and $x(m+2)$ are stored in two delay registers. The Farrow coefficient sub-filters compute $c_0$, $c_1$, and $c_2$ using constant multipliers (0.5 and 1.5) and adders/subtractors. The output is then evaluated as $\hat{x}[m+\mu] = c_0 + \mu(c_1 + \mu c_2)$ using two BF16 multipliers for $\mu$.

    \begin{figure}[htbp]
        \centering
        \includegraphics[width=0.75\textwidth]{../diagram/Q5/first_ver.png}
        \caption{First version block diagram (Farrow structure)}
        \label{fig:first_ver}
    \end{figure}

    Figure~\ref{fig:hw_sharing_ver} shows the hardware sharing version. By reordering computations, the two separate coefficient paths that share common sub-expressions are merged, reducing the total number of adders. The red path highlights the critical path, which runs from the input registers through the shared adder tree and the two sequential $\mu$ multiplications to the output.

    \begin{figure}[htbp]
        \centering
        \includegraphics[width=0.75\textwidth]{../diagram/Q5/hw_sharing_ver.png}
        \caption{Hardware sharing version block diagram with critical path (red)}
        \label{fig:hw_sharing_ver}
    \end{figure}
